NOT NULL

INTRO

                  Ensures a column cannot have NULL values.
                  Used when a field must always contain data.

SQL

                  CREATE TABLE students (
                    id INT,
                    name VARCHAR(50) NOT NULL
                  );

Explanation

                  Name field must always have a value.

OUTPUT

                  Table created with NOT NULL constraint.

UNIQUE

INTRO

                  Ensures all values in a column are different.
                  Duplicates are not allowed.

SQL

                  CREATE TABLE students (
                    id INT UNIQUE,
                    name VARCHAR(50)
                  );

Explanation

                  No two students can have the same ID.

OUTPUT

                  Table created with UNIQUE constraint.

PRIMARY KEY

INTRO

                  Combines NOT NULL + UNIQUE.
                  Used to uniquely identify each record in a table.

SQL

                  CREATE TABLE students (
                    id INT PRIMARY KEY,
                    name VARCHAR(50)
                  );

Explanation

                  ID must be unique and never NULL.

OUTPUT

                  Table created with PRIMARY KEY constraint.

FOREIGN KEY

INTRO

                  Links two tables together.
                  Ensures the referenced value exists in the parent table.

SQL

                  CREATE TABLE marks (
                    id INT,
                    student_id INT,
                    FOREIGN KEY (student_id) REFERENCES students(id)
                  );

Explanation

                  Only allows values in 'student_id' that exist in 'students.id'.

OUTPUT

                  Table created with FOREIGN KEY constraint.

CHECK

INTRO

                  Limits the values that can be placed in a column.
                  Validates data before insertion.

SQL

                  CREATE TABLE students (
                    id INT,
                    marks INT CHECK (marks >= 0 AND marks <= 100)
                  );

Explanation

                  Marks must be between 0 and 100.

OUTPUT

                  Table created with CHECK constraint.

DEFAULT

INTRO

                  Sets a default value for a column when no value is specified.

SQL

                  CREATE TABLE students (
                    id INT,
                    status VARCHAR(20) DEFAULT 'active'
                  );

Explanation

                  If status is not specified, it will default to 'active'.

OUTPUT

                  Table created with DEFAULT constraint.
