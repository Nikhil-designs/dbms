INNER JOIN

INTRO

              Returns only the rows where there's a match in both tables.

SQL

              SELECT students.name, marks.score
              FROM students
              INNER JOIN marks ON students.id = marks.student_id;

Explanation

              Only students with marks will be shown.

OUTPUT

              | name    | score |
              |---------|-------|
              | Alice   | 88    |
              | Bob     | 92    |

RIGHT JOIN

INTRO

              Returns all rows from the right table, even if there's no match on the left.

SQL

              SELECT students.name, marks.score
              FROM students
              RIGHT JOIN marks ON students.id = marks.student_id;

Explanation

              All marks will be shown, even if no matching student exists.

OUTPUT

              | name    | score |
              |---------|-------|
              | Alice   | 88    |
              | Bob     | 92    |
              | NULL    | 75    |

LEFT JOIN

INTRO

              Returns all rows from the left table, even if there's no match on the right.

SQL

              SELECT students.name, marks.score
              FROM students
              LEFT JOIN marks ON students.id = marks.student_id;

Explanation

              All students will be listed, even if they have no marks.

OUTPUT

              | name    | score |
              |---------|-------|
              | Alice   | 88    |
              | Bob     | 92    |
              | Charlie | NULL  |

FULL OUTER JOIN

INTRO

              Returns all rows when there is a match in one of the tables.

SQL

              SELECT students.name, marks.score
              FROM students
              FULL OUTER JOIN marks ON students.id = marks.student_id;

Explanation

              Combines results of LEFT and RIGHT JOIN.

OUTPUT

              | name    | score |
              |---------|-------|
              | Alice   | 88    |
              | Bob     | 92    |
              | Charlie | NULL  |
              | NULL    | 75    |

SELF JOIN

INTRO

              A SELF JOIN is a regular join but the table is joined with itself.
              Used to compare rows in the same table.

SQL

              SELECT A.name AS Employee, B.name AS Manager
              FROM employees A
              JOIN employees B ON A.manager_id = B.id;

Explanation

              We alias the same table as A and B to show which employee reports to which manager.

OUTPUT

              | Employee | Manager  |
              |----------|----------|
              | John     | Alice    |
              | Steve    | Alice    |
              | Mary     | John     |

CROSS JOIN

INTRO

              A CROSS JOIN returns the Cartesian product of two tables.
              Every row of the first table joins with every row of the second.

SQL

              SELECT students.name, subjects.subject
              FROM students
              CROSS JOIN subjects;

Explanation

              If students has 3 rows and subjects has 2 rows, the result will have 6 rows.

OUTPUT

              | name   | subject  |
              |--------|----------|
              | Alice  | Math     |
              | Alice  | Science  |
              | Bob    | Math     |
              | Bob    | Science  |
              | Carol  | Math     |
              | Carol  | Science  |

UNION

INTRO

              Combines result sets of two queries and removes duplicates.

SQL

              SELECT city FROM students
              UNION
              SELECT city FROM teachers;

Explanation

              All unique cities from both students and teachers.

OUTPUT

              | city     |
              |----------|
              | Mumbai   |
              | Delhi    |
              | Chennai  |

UNION ALL

INTRO

              Same as UNION, but includes duplicate rows.

SQL

              SELECT city FROM students
              UNION ALL
              SELECT city FROM teachers;

Explanation

              Duplicates are not removed.

OUTPUT

              | city     |
              |----------|
              | Mumbai   |
              | Delhi    |
              | Mumbai   |

INTERSECT

INTRO

              Returns rows that exist in both result sets.

SQL

              SELECT city FROM students
              INTERSECT
              SELECT city FROM teachers;

Explanation

              Only cities present in both tables.

OUTPUT

              | city     |
              |----------|
              | Mumbai   |

EXCEPT/MINUS

INTRO

              Returns rows from the first query that are not in the second.
              (MySQL uses EXCEPT; Oracle uses MINUS)

SQL

              SELECT city FROM students
              EXCEPT
              SELECT city FROM teachers;

Explanation

              Only cities that belong to students and not teachers.

OUTPUT

              | city     |
              |----------|
              | Chennai  |
