Exercise 1 - Introduction 

  PL/SQL stands for Procedural Language extensions to SQL. 
  It is Oracle Corporation's procedural extension for SQL, and it's used to write programs, 
  procedures, functions, and triggers that interact with an Oracle database.
  
  ✦ Key Features of PL/SQL
  > Tightly integrated with SQL: You can embed SQL statements directly in PL/SQL code.
  > Procedural capabilities: Includes programming constructs like loops, conditions, and exception handling.
  > Modular structure: Allows creation of procedures, functions, packages, and triggers.
  > Error handling: Built-in mechanism to catch and manage runtime errors (exceptions).
  > High performance: Reduces client-server interaction by running logic directly in the database.

Exercise 2 - Block structure 
        ✦A PL/SQL program is written in blocks. Each block has three main parts:
          >Declaration Section: This section starts with the keyword DECLARE. It is an optional section and 
          defines all variables, cursors, sub programs, and other elements to be used in the program.
          
          >Executable Commands: This section is enclosed between the keywords BEGIN and END. 
          It is a mandatory section. It consists of the executable PL/SQL statements of the program.
          
          >Exception Handling : This section starts with the keyword EXCEPTION. 
          This optional section contains exception(s) that handle errors in the program.
          

            
          
✦Syntax:
  DECLARE
     -- Declaration of variables, types, cursors, etc.
  BEGIN
     -- Executable statements (SQL + procedural logic)
  EXCEPTION
     -- Error handling code
END;

✦Example:
  DECLARE 
    message  varchar2(20):= 'Hello, World!'; 
  BEGIN 
    dbms_output.put_line(message); 
  END; 
  /
Exercise 3 - PL/SQL Architecture 
PL/SQL is tightly integrated with the Oracle Database Engine. 
When you write and run PL/SQL code, it goes through multiple layers to execute efficiently.

➜Core Components of PL/SQL Architecture
1. PL/SQL Engine
This is where your PL/SQL block is interpreted and executed.
It handles the procedural parts (like loops, conditions).
Resides either in the client tool (like SQL*Plus) or inside the Oracle DB server.

2. SQL Engine
Executes the SQL statements inside your PL/SQL block (like SELECT, INSERT, etc).
Lives entirely in the Oracle Database.

3. Memory Structures
PGA (Program Global Area): Stores session-specific info like variables.
SGA (System Global Area): Used for shared info like SQL cache, buffers.

4. DBMS_OUTPUT Buffer
Temporarily holds output from DBMS_OUTPUT.PUT_LINE before it's displayed.
