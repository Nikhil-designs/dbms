TRIGGERS

INTRO

            A TRIGGER is a stored block of code that runs automatically when
            a specific event (INSERT, UPDATE, DELETE) occurs on a table.
            
            They are great for enforcing rules, logging, or auto-updating data.
            
            Types:
            - BEFORE / AFTER [INSERT | UPDATE | DELETE]
            - FOR EACH ROW or statement-level
            
            Syntax:
            CREATE OR REPLACE TRIGGER trigger_name
            {BEFORE | AFTER} {INSERT | UPDATE | DELETE}
            ON table_name
            [FOR EACH ROW]
            BEGIN
               -- actions
            END;

PL/SQL

            -- This trigger logs every new insert into the 'students' table
            
            CREATE OR REPLACE TRIGGER trg_student_insert
            AFTER INSERT ON students
            FOR EACH ROW
            BEGIN
               INSERT INTO student_log (stu_id, action, action_time)
               VALUES (:NEW.id, 'INSERTED', SYSDATE);
            END;

Explanation

            - When a new row is inserted into the 'students' table,
              this trigger automatically adds an entry to the 'student_log' table.
            
            - :NEW.id refers to the ID of the newly inserted student.
            - 'student_log' must be a valid table with columns: stu_id, action, action_time.

DROP TRIGGER

INTRO

                      You can drop (delete) a trigger permanently using the DROP TRIGGER statement.
                      Be sure it's no longer needed!

PL/SQL

                      DROP TRIGGER trg_after_row_students;

Explanation

                      This removes the specified trigger from the database.
                      You won’t be able to reuse it unless you recreate it.

OUTPUT

                      Trigger dropped.

OUTPUT

            -- Trigger doesn't return output directly.
            -- But inserting a student like:
            
            INSERT INTO students (id, name) VALUES (201, 'Neo');
            
            -- Would auto-generate a log like:
            | stu_id | action   | action_time         |
            |--------|----------|---------------------|
            | 201    | INSERTED | 2025-04-19 11:00:00 |

BEFORE STATEMENT-LEVEL TRIGGER

INTRO

              Fires once before the actual statement (INSERT/UPDATE/DELETE),
              not for each row. Good for validations, logging, or setting session variables.

PL/SQL

              CREATE OR REPLACE TRIGGER trg_before_insert_students
              BEFORE INSERT ON students
              BEGIN
                 DBMS_OUTPUT.PUT_LINE('Preparing to insert into students table...');
              END;

Explanation

              This will run once before an INSERT statement starts,
              regardless of how many rows are inserted.

OUTPUT

              Preparing to insert into students table...

AFTER STATEMENT-LEVEL TRIGGER

INTRO

              Fires once after a DML operation completes.
              Use it to log completion or trigger follow-up actions.

PL/SQL

              CREATE OR REPLACE TRIGGER trg_after_update_students
              AFTER UPDATE ON students
              BEGIN
                 DBMS_OUTPUT.PUT_LINE('Student table has been updated.');
              END;

Explanation

              This will print a message after the UPDATE statement finishes,
              no matter how many rows were updated.

OUTPUT

              Student table has been updated.

BEFORE ROW-LEVEL TRIGGER

INTRO

              Fires once for every row affected, before the change is made.
              Great for value manipulation or data normalization.

PL/SQL

              CREATE OR REPLACE TRIGGER trg_before_row_students
              BEFORE INSERT ON students
              FOR EACH ROW
              BEGIN
                 :NEW.name := INITCAP(:NEW.name); -- Capitalize name
              END;

Explanation

              This will modify the name to "Proper Case" before it's saved in the DB.

OUTPUT

              INSERT INTO students (id, name) VALUES (1, 'neo');
              -- Triggers BEFORE ROW, changes 'neo' ➝ 'Neo'

AFTER ROW-LEVEL TRIGGER

INTRO

              Fires after each row is modified.
              Good for logging, syncing, or audit trails.

PL/SQL

              CREATE OR REPLACE TRIGGER trg_after_row_students
              AFTER DELETE ON students
              FOR EACH ROW
              BEGIN
                 INSERT INTO student_log (stu_id, action, action_time)
                 VALUES (:OLD.id, 'DELETED', SYSDATE);
              END;

Explanation

              Each time a row is deleted, a log entry is added with the student's ID.

OUTPUT

              DELETE FROM students WHERE id = 2;
              -- Auto-log:
              | stu_id | action  | action_time         |
              |--------|---------|---------------------|
              | 2      | DELETED | 2025-04-19 13:00:00 |
