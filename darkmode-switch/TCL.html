COMMIT

INTRO

              COMMIT saves all the changes made during the current transaction permanently.

SQL

              UPDATE students SET marks = 95 WHERE name = 'Neo';
              COMMIT;

Explanation

              Once COMMIT is executed, changes can't be undone using ROLLBACK.

OUTPUT

              Query OK, 1 row affected
              Transaction committed successfully.

ROLLBACK

INTRO

              ROLLBACK undoes changes made in the current transaction.

SQL

              DELETE FROM students WHERE name = 'Trinity';
              ROLLBACK;

Explanation

              The delete operation is canceled. Trinity is safe!

OUTPUT

              Query OK, 1 row affected
              Transaction rolled back successfully.

GRANT

INTRO

              GRANT is used to give user access privileges to the database.

SQL

              GRANT SELECT, INSERT ON students TO 'agent'@'localhost';

Explanation

              Allows 'agent' user to view and insert data into the students table.

OUTPUT

              Privileges granted.

REVOKE

INTRO

              REVOKE removes user privileges previously granted.

SQL

              REVOKE INSERT ON students FROM 'agent'@'localhost';

Explanation

              Now 'agent' can no longer insert data into the students table.

OUTPUT

              Privileges revoked.
