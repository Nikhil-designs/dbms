Exercise 1 - Introduction 

  SQL stands for Structured Query Language. It is the standard language used to manage and 
  manipulate relational databases. With SQL, you can perform tasks such as:
  ✦Inserting data into tables
  ✦Querying specific data using conditions
  ✦Updating existing records
  ✦Deleting data
  ✦Creating and modifying tables and databases
  ➜ Why is SQL Important?
     SQL is essential because it allows users to interact with large datasets in a precise, readable, and 
     efficient way. Whether you're a backend developer, data analyst, or just curious about data, 
     SQL is the foundation of working with structured data.

Exercise 2 - SQL Commands 
          SQL commands are the instructions we give to the database to perform specific tasks. These
          commands are grouped into five main categories, each with its own role in database management.

          1. DDL ➜ Data Definition Language
          Used to define and modify the structure of database objects like tables and schemas.
          2. DML ➜ Data Manipulation Language
          Used to manipulate data within tables.
          3. DQL ➜ Data Query Language
          Used to query data from the database.
          4. DCL ➜ Data Control Language
          Used to control access to data.
          5. TCL ➜ Transaction Control Language
          Used to manage transactions and control the execution of SQL statements.
Exercise 3 - SQL DataTypes 
          SQL data types define the kind of data that can be stored in a column of a table. 
          Choosing the right data type ensures efficient storage and 
          accurate data representation.

          1. Numeric Types
          
          Used to store numbers — both whole and decimal.
          ✦INT ➜ Whole numbers (e.g., 5, 42)          
          ✦SMALLINT, BIGINT ➜ Smaller or larger ranges of integers          
          ✦DECIMAL(p, s) ➜ Fixed-point number (e.g., DECIMAL(5,2) stores 999.99)          
          ✦FLOAT, REAL ➜ Approximate floating-point numbers
          
          2. Character/String Types
          
          Used to store text.
          ✦CHAR(n) ➜ Fixed-length string (e.g., CHAR(10) stores exactly 10 characters)          
          ✦VARCHAR(n) ➜ Variable-length string (e.g., names, email addresses)
          ✦TEXT ➜ Long-form text (e.g., articles, descriptions)
          
          3. Date and Time Types
          
          Used to store dates and timestamps.
          ✦DATE ➜ Stores date (YYYY-MM-DD)
          ✦TIME ➜ Stores time (HH:MM:SS)
          ✦DATETIME / TIMESTAMP ➜ Stores full date and time
          
          4. Boolean Type:
          
          Used to store true/false values.
          ✦BOOLEAN ➜ Typically stores TRUE or FALSE (may be stored as 1 or 0 internally)
Exercise 4 - Pros 
          1. Easy to Learn & Use:
          SQL uses simple English-like syntax, making it beginner-friendly.

          2. Efficient Data Retrieval:
          You can quickly query large amounts of data using SELECT, JOIN, WHERE, etc.

          3. Standardized Language:
          SQL is an ANSI and ISO standard, supported by most relational database systems 
          (MySQL, PostgreSQL, SQL Server, Oracle, etc.).

          4. Performs Multiple Operations:
          With SQL, you can create, read, update, and delete (CRUD) data, all using consistent syntax.

          5. Data Security & Access Control:
          You can manage user permissions using GRANT and REVOKE.

          6. Handles Complex Queries:
          SQL supports subqueries, nested queries, views, and stored procedures for advanced logic.

          7. Supports Integration:
          SQL works well with various programming languages and applications (like Python, Java, PHP, etc.).

          8. Data Manipulation & Analysis:
          Perfect for aggregating, filtering, and analyzing data using functions 
          like SUM(), COUNT(), GROUP BY, and ORDER BY.
