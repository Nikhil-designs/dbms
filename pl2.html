IMPLICIT CURSOR

INTRO

PL/SQL uses implicit cursors for SQL statements that return only one row.
They’re automatically managed and useful for quick operations.
            
Syntax:
    -- Oracle manages cursor automatically
    -- You can check attributes using SQL%
        
Example Attributes:
    SQL%FOUND – TRUE if at least one row was affected
    SQL%NOTFOUND – TRUE if no rows affected
    SQL%ROWCOUNT – Number of rows affected

PL/SQL

BEGIN
    UPDATE students
    SET name = 'Updated'
    WHERE id = 101;

    IF SQL%FOUND THEN
        DBMS_OUTPUT.PUT_LINE(SQL%ROWCOUNT || ' row(s) updated.');
    ELSE
        DBMS_OUTPUT.PUT_LINE('No student found with ID 101.');
        END IF;
    END;

Explanation

After updating, PL/SQL uses the implicit cursor to check if rows were affected.
SQL%ROWCOUNT returns how many rows were updated.

OUTPUT

1 row(s) updated.
    -- or --
No student found with ID 101.

EXPLICIT CURSOR

INTRO

An explicit cursor is used when you need to process multiple rows from a query,
giving you more control with OPEN, FETCH, and CLOSE steps.

Syntax:
DECLARE
   CURSOR c_name IS SELECT_statement;
BEGIN
   OPEN c_name;
   LOOP
      FETCH c_name INTO variable;
      EXIT WHEN c_name%NOTFOUND;
      -- Do something
   END LOOP;
   CLOSE c_name;
END;

PL/SQL

DECLARE
   CURSOR stu_cursor IS
      SELECT id, name FROM students;
   stu_id students.id%TYPE;
   stu_name students.name%TYPE;
BEGIN
   OPEN stu_cursor;
   LOOP
      FETCH stu_cursor INTO stu_id, stu_name;
      EXIT WHEN stu_cursor%NOTFOUND;
      DBMS_OUTPUT.PUT_LINE('ID: ' || stu_id || ', Name: ' || stu_name);
   END LOOP;
   CLOSE stu_cursor;
END;

Explanation

The cursor processes each student’s ID and name, printing it one at a time.
The loop exits when all rows are fetched.

OUTPUT

ID: 101, Name: Alice
ID: 102, Name: Bob
ID: 103, Name: Charlie

EXCEPTION

INTRO

Exceptions in PL/SQL handle errors that occur during execution.
Use the EXCEPTION block to catch and respond to errors gracefully.

Syntax:
BEGIN
   -- statements
EXCEPTION
   WHEN exception_type THEN
      -- handler statements
END;

PL/SQL

BEGIN
   DBMS_OUTPUT.PUT_LINE(10 / 0); -- Division by zero
EXCEPTION
   WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Cannot divide by zero!');
END;

Explanation

If an error occurs (like dividing by 0), the exception block catches it
and displays a friendly message.

OUTPUT

Cannot divide by zero!

PREDEFINED EXCEPTION

INTRO

PL/SQL provides many built-in (predefined) exceptions like:
- NO_DATA_FOUND
- TOO_MANY_ROWS
- ZERO_DIVIDE

These help you catch common runtime issues easily.

PL/SQL

DECLARE
   stu_name students.name%TYPE;
BEGIN
   SELECT name INTO stu_name FROM students WHERE id = 999; -- Assuming ID 999 doesn't exist
   DBMS_OUTPUT.PUT_LINE('Student: ' || stu_name);
EXCEPTION
   WHEN NO_DATA_FOUND THEN
      DBMS_OUTPUT.PUT_LINE('No student found with this ID.');
END;

Explanation

When a SELECT INTO fails to find a row, NO_DATA_FOUND is triggered.
This lets us display a custom message instead of crashing.

OUTPUT

No student found with this ID.

USER-DEFINED EXCEPTION

INTRO

You can create your own exceptions using:
- DECLARE exception_name EXCEPTION;
- RAISE exception_name;

Useful for business rules or custom validations.

PL/SQL

DECLARE
   age NUMBER := 15;
   underage EXCEPTION;
BEGIN
   IF age < 18 THEN
      RAISE underage;
   ELSE
      DBMS_OUTPUT.PUT_LINE('Eligible for registration.');
   END IF;
EXCEPTION
   WHEN underage THEN
      DBMS_OUTPUT.PUT_LINE('Too young to register!');
END;

Explanation

We define a custom rule: age must be 18+.
If not, we RAISE a custom exception and catch it in the EXCEPTION block.

OUTPUT

Too young to register!
